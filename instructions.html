Instruction Reference

Memory Access Instructions

LOAD, LOADS

Load a value from memory into a register.

Arguments

  • register (destination), immediate (source)
  • register (destination), immediate indirect (source)
  • register (destination), register indirect (source)

Flags Set

sign, zero

Size Modifiers

#b (byte), #h (half word, two bytes), #w (word, four bytes)

Notes

  • LOAD: values are interpreted as unsigned. If a value is smaller than 4 bytes (i.e. the #b or #h size modifiers are used) then the values will be zero padded to 32 bits.
  • LOADS: values are interpreted as signed. If a value is smaller than 4 bytes (i.e. the #b or #h size modifiers are used) then the values will be sign extended to 32 bits.

Examples

// Load the value 123 into register R1.
LOAD R1 123 // Load the four byte value at address 123 into register R1.
LOAD R1 @123 // Load the one byte value at address 0xA0 into register R1.
LOAD#b R1 @0xA0 // Load the four byte value at the address contained in R2 into
// register R1.
LOAD R1 @R2

STORE

Store a register value in memory.

Arguments

  • immediate indirect (source), register (destination)
  • register indirect (source), register (destination)

Flags Set

None

Size Modifiers

#b (byte), #h (half word, two bytes), #w (word, four bytes)

Examples

// Store the four bytes in R1 to address 123.
STORE @123 R1 // Store the lowest byte in R1 to address contained in R2.
STORE#b @R2 R1

Arithmetic Instructions

ADD

Add two register values.

Arguments

  • register (sum), register (addend), register (addend)

Flags Set

carry, overflow, sign, zero

Notes

Binary addition is the same for signed and unsigned numbers.

Examples

LOAD R1 13
LOAD R2 17
// Add the values in R1 and R2 and store the sum in R3.
ADD R3 R1 R2
// R3 is now 30.

SUB

Subtract one register value from another.

Arguments

  • register (difference), register (minuend), register (subtrahend)

Flags Set

carry, overflow, sign, zero

Notes

Binary subtraction is the same for signed and unsigned numbers.

Examples

LOAD R1 13
LOAD R2 17
// Subtract R2 from R1 and store the difference in R3.
SUB R3 R1 R2
// R3 is now -4

NEG

Negate the value of a register.

Arguments

  • register (destination), register (source)

Flags Set

carry, overflow, sign, zero

Notes

This is the same as subtracting from zero.

Examples

LOAD R1 5
// Negate R1 and store the result in R2.
NEG R2 R1
// R2 is now -5

CMP

Compare two register values.

Arguments

  • register (minuend), register (subtrahend)

Flags Set

carry, overflow, sign, zero

Notes

This is the same as SUB, but the difference is discarded, so only flags are changed.

Examples

// Compare the value of R1 to the value of R2.
CMP R1 R2

MUL, MULS

Multiply two register values.

Arguments

  • register (product), register (factor), register (factor)

Flags Set

carry, overflow, sign, zero

Notes

  • MUL: performs unsigned multiplication. Values are interpreted as unsigned.
  • MULS: performs signed multiplication. Values are interpreted as signed.
  • The carry and overflow flags will be set if the product cannot be represented with 32 bits.

Examples

LOAD R1 13
LOAD R2 17
// Multiply R1 with R2 and store the result in R3.
MUL R3 R1 R2
// R3 is now 221.

DIV, DIVS

Divide a register value by another.

Arguments

  • register (quotient), register (remainder), register (dividend), register (divisor)

Flags Set

carry, overflow, sign, zero

Notes

  • DIV: performs unsigned division. Values are interpreted as unsigned. carry and overflow flags will always be unset after the instruction executes.
  • DIVS: performs signed division. Values are interpreted as signed. If the quotient cannot be represented with 32 bits (only occurs when the 32 bit minimum value -2,147,483,648 is divided by -1) then the carry and overflow flags will be set.
  • If a divide by zero occurs then the CPU will halt. In the future this will be changed so a CPU exception occurs.

Examples

LOAD R1 27
LOAD R2 4
// Divide R1 by R2, store the result in R3 and the remainder in R4.
DIV R3 R4 R1 R2
// R3 is now 6 and R4 is now 3. // Divide R1 by R2, store the result in R3 and discard the remainder
// by writing to R0.
DIV R3 R0 R1 R2

Bitwise Instructions

AND

Compute the bitwise AND of two registers.

Arguments

  • register (destination), register (source), register (source)

Flags Set

carry, overflow, sign, zero

Notes

This instruction will always clear the carry and overflow flags.

Examples

LOAD R1 4
LOAD R2 7
// AND R1 and R2 and store the result in R3.
AND R3 R1 R2
// R3 is now 4.

OR

Compute the bitwise OR of two registers.

Arguments

  • register (destination), register (source), register (source)

Flags Set

carry, overflow, sign, zero

Notes

This instruction will always clear the carry and overflow flags.

Examples

LOAD R1 3
LOAD R2 5
// OR R1 and R2 and store the result in R3.
OR R3 R1 R2
// R3 is now 7

XOR

Compute the bitwise exclusive OR of two registers.

Arguments

  • register (destination), register (source), register (source)

Flags Set

carry, overflow, sign, zero

Notes

This instruction will always clear the carry and overflow flags.

Examples

LOAD R1 3
LOAD R2 5
// XOR R1 and R2 and store the result in R3.
XOR R3 R1 R2
// R3 is now 6

NOT

Compute the bitwise negation of a register.

Arguments

  • register (destination), register (source)

Flags Set

carry, overflow, sign, zero

Notes

This instruction will always clear the carry and overflow flags.

Examples

LOAD R1 0x88888888
// NOT R1 and store the result in R2.
NOT R2 R1
// R2 is now 0x77777777.

SHR, SHRS

Bitwise shift the value of a register right.

Arguments

  • register (destination), register (source), register (shift count)

Flags Set

carry, overflow, sign, zero

Notes

  • SHR: zeros will be shifted into the most significant bit (leftmost bit) for each shift.
  • SHRS: shifted in bits will match the most significant bit, i.e. if the MSB is 1 then the shifted in bit will be 1, otherwise it will be 0.
  • The carry flag will be set if the last shifted out bit was set.
  • These instructions will always clear the overflow flag.
  • If the shift count register is greater than 31 then the value will be shifted by the shift count modulo 32.

Examples

LOAD R1 12
LOAD R2 3
// Shift R1 right R2 times and store the result in R3.
SHR R3 R1 R2
// R3 is now 1 and the carry flag is set.

SHL

Bitwise shift the value of a register left.

Arguments

  • register (destination), register (source), register (shift count)

Flags Set

carry, overflow, sign, zero

Notes

  • Zeros will be shifted into the least significant bit (rightmost bit) for each shift. If the third register (shift count) is greater than 31 then the value will be shifted by the shift count modulo 32.
  • The carry flag will be set if the last shifted out bit was set.
  • This instruction will always clear the overflow flag.

Examples

LOAD R1 1
LOAD R2 3
// Shift R1 left R2 times and store the result in R3.
SHL R3 R1 R2
// R3 is now 8.

Branch Instructions

JMP

Unconditionally jump execution to an address.

Arguments

  • immediate (destination)
  • register (destination)

Flags Set

None

Examples

// Execute the next instruction at address pointed to by label "foobar".
JMP foobar // Execute the next instruction at address 123.
JMP 123 // Execute the next instruction at the address contained in R1.
JMP R1

BE

Branch execution if equal.

Arguments

  • immediate (destination)
  • register (destination)

Flags Set

None

Notes

The branch will be taken if the zero flag is set.

Examples

// Execute the next instruction at address pointed to by label
// "foobar" if R1 and R2 are equal.
CMP R1 R2
BE foobar // Execute the next instruction at address 123 if R1 and R2 are equal.
CMP R1 R2
BE 123 // Execute the next instruction at the address contained in R3 if R1
// and R2 are equal.
CMP R1 R2
BE R3

BNE

Branch execution if not equal.

Arguments

  • immediate (destination)
  • register (destination)

Flags Set

None

Notes

The branch will be taken if the zero flag is not set.

Examples

// Execute the next instruction at address pointed to by label
// "foobar" if R1 and R2 are not equal.
CMP R1 R2
BNE foobar // Execute the next instruction at address 123 if R1 and R2 are not
// equal.
CMP R1 R2
BNE 123 // Execute the next instruction at the address contained in R3 if R1
// and R2 are not equal.
CMP R1 R2
BNE R3

BG, BGS

Branch execution if greater.

Arguments

  • immediate (destination)
  • register (destination)

Flags Set

None

Notes

  • BG: interprets flags as if unsigned values were compared. The branch will be taken if the carry and zero flags are not set.
  • BGS: interprets flags as if signed values were compared. The branch will be taken if the overflow and sign flags do not match, and the zero flag is not set.

Examples

// Execute the next instruction at address pointed to by label
// "foobar" if R1 is greater than R2.
CMP R1 R2
BG foobar // Execute the next instruction at address 123 if R1 is greater
// than R2.
CMP R1 R2
BG 123 // Execute the next instruction at the address contained in R3 if R1
// is greater than R2.
CMP R1 R2
BG R3

BGE, BGES

Branch execution if greater or equal.

Arguments

  • immediate (destination)
  • register (destination)

Flags Set

None

Notes

  • BGE: interprets flags as if unsigned values were compared. The branch will be taken if the carry flag is not set.
  • BGES: interprets flags as if signed values were compared. The branch will be taken if the overflow and sign flags match.

Examples

// Execute the next instruction at address pointed to by label
// "foobar" if R1 is greater than or equal to R2.
CMP R1 R2
BGE foobar // Execute the next instruction at address 123 if R1 is greater than
// or equal to R2.
CMP R1 R2
BGE 123 // Execute the next instruction at the address contained in R3 if R1
// is greater than or equal to R2.
CMP R1 R2
BGE R3

BL, BLS

Branch execution if less.

Arguments

  • immediate (destination)
  • register (destination)

Flags Set

None

Notes

  • BL: interprets flags as if unsigned values were compared. The branch will be taken if the carry flag is set.
  • BLS: interprets flags as if signed values were compared. The branch will be taken if the overflow and sign flags do not match.

Examples

// Execute the next instruction at address pointed to by label
// "foobar" if R1 is less than R2.
CMP R1 R2
BL foobar // Execute the next instruction at address 123 if R1 is less than R2.
CMP R1 R2
BL 123 // Execute the next instruction at the address contained in R3 if R1
// is less than R2.
CMP R1 R2
BL R3

BLE, BLES

Branch execution if less than or equal.

Arguments

  • immediate (destination)
  • register (destination)

Flags Set

None

Notes

  • BLE: interprets flags as if unsigned values were compared. The branch will be taken if the carry or zero flags are set.
  • BLES: interprets flags as if signed values were compared. The branch will be taken if the overflow and sign flags do not match, or the zero flag is set.

Examples

// Execute the next instruction at address pointed to by label
// "foobar" if R1 is less than or equal to R2.
CMP R1 R2
BLE foobar // Execute the next instruction at address 123 if R1 is less than or
// equal to R2.
CMP R1 R2
BLE 123 // Execute the next instruction at the address contained in R3 if R1

CMP R1 R2

Stack Instructions

PUSH

Push a register value on the stack.

Arguments

  • register (source)

Flags Set

None

Notes

Decrements SP by four (the size of one word) and stores the value in the source register to the address pointed to by SP.

Examples

LOAD SP 0xFFFF0
LOAD R1 0xDEADBEEF
// Subtract 4 from SP and write R1 to the address contained in SP.
PUSH R1
// SP is now 0xFFFEC and 0xDEADBEEF is written at that address (in
// little-endian order).

POP

Pop a value off the stack into a register.

Arguments

  • register (destination)

Flags Set

None

Notes

Loads a word from the address contained in SP into the destination register and then increments SP by four (the size of one word).

Examples

LOAD SP 0xFFFF0
LOAD R1 0xDEADBEEF
PUSH R1
POP R2
// R2 is now contains 0xDEADBEEF.

CALL

Store the program counter and jump to an address.

Arguments

  • immediate (destination)
  • register (destination)

Flags Set

None

Notes

This instruction is the same as PUSH PC followed by JMP destination.

Examples

// Program that prints "ABC".

start:
LOAD R1 'A'
PRINTC R1
CALL print_b
LOAD R1 'C'
PRINTC R1
HALT

print_b:
LOAD R1 'B'
PRINTC R1
RET

RET

Read an address off the stack and jump to it.

Arguments

None

Flags Set

None

Notes

This instruction is the same as POP PC.

Examples

// Program that prints "ABC".

start:
LOAD R1 'A'
PRINTC R1
CALL print_b
LOAD R1 'C'
PRINTC R1
HALT

print_b:
LOAD R1 'B'
PRINTC R1
RET

Print Instructions

PRINTC

Print an ASCII character to the screen.

Arguments

  • register (character value)

Flags Set

None

Examples

// Print the character "A" to the screen.
LOAD R1 'A'
PRINTC R1 // Print the character "B" to the screen.
LOAD R1 66
PRINTC R1

PRINTI

Print an integer to the screen.

Arguments

  • register (value)

Flags Set

None

Examples

// Print the number 123 to the screen.
LOAD R1 123
PRINTI R1 // Print the number -456 to the screen.
LOAD R1 -456
PRINTI R1
